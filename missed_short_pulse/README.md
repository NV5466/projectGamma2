# missed_short_pulse

Discrete-input timing seed folder for cases where an electrical pulse exists, but the scan, filtering, acquisition, or software path misses or misrepresents it.

This seed must stay distinct from high-speed input bounce. Missed short pulse is about the path failing to preserve a real pulse; high-speed input bounce is about repeated input events creating a repeatable middle-node response that can corrupt downstream interpretation.

Boundary: synthetic validation only; no field calibration claim.
